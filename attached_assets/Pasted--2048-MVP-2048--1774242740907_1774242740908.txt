모바일 우선 반응형 웹앱으로 실행되는 2048 변형 게임 MVP를 만들어줘.

프로젝트 목표:
- 캐릭터 이미지 없이 "색상 단계"만으로 플레이 가능한 2048 스타일 퍼즐 게임
- 귀엽고 부드러운 느낌의 UI
- 빠르게 테스트 가능한 웹 MVP
- 추후 식물/동물/날씨/풍경 테마로 확장 가능한 구조
- 추후 광고 SDK를 쉽게 붙일 수 있도록 광고 자리와 광고 로직 구조를 미리 포함

기술 요구사항:
- HTML, CSS, JavaScript만 사용
- 별도 백엔드 없이 프론트엔드만으로 동작
- 하나의 Replit 프로젝트에서 바로 실행 가능
- 모바일 브라우저에 최적화
- 터치 스와이프와 키보드 방향키 모두 지원
- 코드 구조는 이해하기 쉽게 분리
- 주석을 충분히 달아줘

핵심 게임 규칙:
- 4x4 보드
- 시작 시 2개의 타일 생성
- 한 번 움직일 때마다 빈 칸에 새 타일 1개 생성
- 같은 단계의 타일이 만나면 다음 단계로 합쳐짐
- 기본 값 구조는 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048
- 화면에는 숫자를 보여주되, 숫자 대신 단계 이름도 같이 표시 가능하게 구조를 만들어줘
- 2048 타일 도달 시 클리어 메시지 표시
- 더 이상 움직일 수 없으면 게임 오버 표시
- 새 게임 버튼 제공
- 현재 점수와 최고 점수(localStorage 저장) 표시

UI/UX 요구사항:
- 전체 분위기는 부드럽고 귀여운 느낌
- 너무 화려하지 않고 미니멀하게
- 카드/타일은 둥근 모서리
- 부드러운 이동/합치기 애니메이션
- 모바일에서 한 손 조작이 편해야 함
- 보드와 버튼이 화면에 꽉 차되 답답하지 않게 여백 유지
- 상단에 게임 제목, 점수, 최고 점수, 새 게임 버튼 배치
- 하단에는 간단한 설명 문구 배치
- 광고 영역이 들어갈 공간도 자연스럽게 포함

색상 테마 요구사항:
- 숫자 대신 색상만으로도 단계 차이가 느껴지게 해줘
- 기본 테마는 식물 느낌의 파스텔 그린 계열
- 단계가 올라갈수록 색이 진해지거나 화려해지게 설계
- 2 / 4 / 8 / 16 / 32 / 64 / 128 / 256 / 512 / 1024 / 2048 각각 다른 배경색 지정
- 텍스트 가독성 고려

확장 가능 구조:
- 이후 식물 / 동물 / 날씨 / 풍경 테마를 추가할 수 있도록
- 색상 정보와 단계 이름을 별도 설정 객체로 분리
- 예시로 "plant" 테마를 기본값으로 넣어줘
- 테마 데이터 구조는 나중에 쉽게 추가 가능해야 함

plant 테마 단계 이름 예시:
- 2: 씨앗
- 4: 싹
- 8: 새싹
- 16: 잎
- 32: 작은 화분
- 64: 꽃봉오리
- 128: 꽃
- 256: 큰 꽃
- 512: 반짝 꽃
- 1024: 희귀 꽃
- 2048: 전설의 꽃

광고 구조 요구사항:
- 실제 광고 SDK는 아직 넣지 말 것
- 대신 광고 자리를 미리 포함한 구조로 만들어줘
- 향후 광고 SDK를 쉽게 교체할 수 있도록 adService.js 또는 유사한 파일로 광고 로직을 분리
- 아래 mock 광고 함수들을 만들어줘:
  - showRewardedAd()
  - showInterstitialAd()
  - showBannerPlaceholder()
- 실제 광고 대신 mock 방식으로 동작하게 해줘
- 보상형 광고는 1.5~2초 정도 로딩 후 성공 콜백을 실행하는 구조로 만들어줘
- 전면 광고도 mock 모달로 표시 후 닫히게 만들어줘
- 배너 광고는 실제 광고 대신 "AD PLACEHOLDER" 박스로 표시해줘

광고 UX 자리 요구사항:
- 상단 또는 하단에 배너 광고 자리 크기의 placeholder 영역 추가
- 게임 오버 모달에 "광고 보고 이어하기" 버튼 추가
- 승리 모달에도 "광고 보고 계속 유리하게 플레이" 같은 확장 버튼 자리 추가 가능하게 구조 설계
- 설정 또는 메뉴 영역에 "광고 제거(향후 유료)" 자리만 추가
- 광고 버튼은 실제로는 mock 광고 함수만 호출하게 해줘

이어하기 보상 규칙:
- 게임 오버 시 1회에 한해 "광고 보고 이어하기" 가능
- 광고 시청 성공 시:
  - 빈 칸 1개 확보 또는
  - 가장 낮은 단계 타일 1개 제거
- 구현은 단순하고 안정적인 방식으로 작성
- 이어하기는 한 판당 1회만 허용

전면 광고 mock 규칙:
- 실제 광고는 아니고
- 3판에 1번 정도 showInterstitialAd()가 호출될 수 있는 구조만 잡아줘
- 지금은 게임 종료 후 조건 충족 시 mock 모달로 표시

추가 기능:
- 사운드는 없어도 됨
- 다크모드는 필요 없음
- localStorage로 최고 점수 저장
- 게임 리셋 시 확인 모달 표시
- 승리 후에도 계속 플레이 가능 버튼 제공
- 향후 광고 SDK를 붙일 수 있도록 광고 관련 함수는 UI와 분리

산출물 요구사항:
1. 실행 가능한 전체 코드 작성
2. 파일 구조 제안
3. 각 파일 역할 설명
4. Replit에서 바로 실행되도록 작성
5. 내가 수정하기 쉬운 포인트 정리
6. 광고 SDK를 나중에 붙일 때 어디를 바꾸면 되는지 설명

구현 안정성 요구사항:
- swipe 입력이 부드럽게 동작하도록 구현
- 같은 턴에는 한 타일이 한 번만 합쳐지게 구현
- 애니메이션 때문에 게임 상태가 꼬이지 않게 안정적으로 작성
- CSS는 깔끔하게 정리
- MVP지만 완성도 있게 보여줘

개발 우선순위:
1. 먼저 게임이 완전히 동작하게 구현
2. 그 다음 UI 정리
3. 그 다음 광고 placeholder와 mock 광고 기능 연결
4. 마지막에 코드 설명과 수정 포인트 정리

추가 요청:
- 먼저 최소 구조로 완성하고, 이후 개선 포인트를 별도 섹션으로 제안해줘
- 한 번에 과하게 복잡하게 만들지 말고 MVP 완성도를 우선해줘
